# Contributing to RAWVE Frontend

First off, thank you for considering contributing to the RAWVE web client! It's people like you that make RAWVE a great platform for the streaming community.

Whether you're fixing a bug, improving the UI, or adding a new feature — your help is greatly appreciated.

---

## 🚀 Getting Started

### 1. Fork and Clone

Fork the repository to your own GitHub account, then clone it locally:

```bash
git clone https://github.com/YOUR_USERNAME/rawve-frontend.git
cd rawve-frontend
```

### 2. Branch Naming

Create a new branch for your work using a descriptive name:

```bash
git checkout -b feature/add-dark-mode-toggle
# or
git checkout -b fix/chat-auto-scroll
```

### 3. Development Setup

Install dependencies and start the development server:

```bash
npm install
npm run dev
```

## 🏗️ Code Guidelines

We follow a strict **Feature-Driven Architecture**. Please adhere to these rules to keep the codebase clean and maintainable.

| Rule | Description |
|---|---|
| **Feature Isolation** | Components specific to a feature (e.g., `ChatBox`) belong in `src/features/chat/components/`, not in the global `src/components/` folder |
| **Global UI** | `src/components/ui/` is strictly reserved for "dumb" UI components (Buttons, Inputs, Dialogs) generated by Shadcn UI — no business logic here |
| **Type Safety** | All props, states, and API responses must be strongly typed in `src/types/` — PRs using `any` will not be accepted |
| **Tailwind Styling** | Use the `cn()` utility from `src/lib/utils.ts` when merging Tailwind classes conditionally |

## 📝 Commit Convention

We follow the [Conventional Commits](https://www.conventionalcommits.org/) specification to help us auto-generate changelogs.

**Format:** `<type>: <description>`

| Prefix | Use for |
|---|---|
| `feat:` | New feature |
| `fix:` | Bug fix |
| `style:` | Formatting, no logic change |
| `refactor:` | Code refactoring |
| `docs:` | Documentation only |
| `test:` | Adding tests |
| `chore:` | Build process or tooling |

**Examples:**

```
feat: add new creator dashboard layout
fix: resolve hydration error on video player
refactor: move chat store to features/chat
chore: update tanstack query dependency
```

---

## 📬 Submitting a Pull Request

Before opening a PR, please make sure:

1. **Builds successfully** — run `npm run build` to check for TypeScript or build errors
2. **Linting passes** — run `npm run lint` and fix any warnings or errors
3. **Screenshots included** — if your PR introduces visual changes, include screenshots or a short GIF in the description
4. **Detailed description** — explain what you changed and why

Once you're ready, push your branch to your fork and open a Pull Request against the `main` branch. A maintainer will review your code as soon as possible!
