## 🔒 Reporting a Vulnerability

If you discover a security vulnerability — such as Cross-Site Scripting (XSS), token leakage, or unauthorized access — please **do not open a public issue**.

Report it privately to the maintainer to give us time to patch it before public disclosure.

## 📋 What to Include in Your Report

When reporting, please provide:

1. **Description** — a clear explanation of the vulnerability
2. **Reproduction steps** — step-by-step instructions to reproduce it
3. **Impact** — the potential impact on users or the platform
4. **Suggested fix** — if you have one (optional but appreciated)

We take security seriously and will respond to your report as quickly as possible.
